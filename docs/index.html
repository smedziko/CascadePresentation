Erklärung:
Cascade Persist & Cascade Remove

  • Cascade ist ein Begriff, der im ORM (Object-Relational Mapping) verwendet wird

  • Das Verhalten beim Speichern (persist) bzw. Löschen (remove) von Objekten mit Beziehungen in einer Datenbank wird beschrieben

  • Automatische Übertragung von Befehlen/Aktionen von einem "Vaterobjekt" auf ein "Kindobjekt"
    • Vorteile:
  • Datenintegrität
  • Vereinfachte Datenbankinteraktion
  • Zeitersparnis
    • Nachteile:
  • Performance
  • Unerwartete Datenänderungen
    • Beispiel: CartProduct.php
    Beispiel: Cart.php
